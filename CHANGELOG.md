- **True offline support via a service worker** (`src/sw.js` → built to `dist/sw.js`): once the app has been opened successfully one time over a secure origin, the home-screen icon keeps working with zero connectivity (airplane mode, no signal) instead of failing to load. Network-first for the app page itself (so an online visit always picks up the latest build, falling back to the last cached copy offline); cache-first-with-background-refresh for everything else (mainly the Google Fonts requests). `scripts/build.js` now also emits `dist/sw.js`, with its cache name versioned by a content hash of the built page so a redeploy with changed content automatically invalidates old caches. **Requires a secure context — HTTPS, or `localhost` for local dev — browsers refuse to register a service worker over plain HTTP on a LAN IP**, so this pairs with a real deployed URL (e.g. the existing GitHub Pages workflow), not the `npm run dev` LAN-testing flow. Verified with Playwright: after exactly one online visit, a fully offline reload still renders the app and the quiz flow works end-to-end.

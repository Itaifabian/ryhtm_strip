- **Reworked the Cases tab** from a curated drug-vignette quiz walkthrough into a single interactive mode: a **diagnostic tree walker** — "walk the workup yourself," not a quiz and not a flat reference. This went through several iterations on user feedback before landing here: a curated drug-vignette quiz walkthrough → a flat browsable situation/treatment reference (`CASE_GUIDES`) → an auto-generated MCQ practice mode (scrapped — wrong shape entirely) → a one-card-at-a-time decision tree → the final left-to-right visual flow below. The flat reference (`CASE_GUIDES`) has been **removed**, per explicit instruction to keep just the decision tree — every fact it held was condensed into the trees instead, so nothing is lost, just restructured.
  - New data file `src/data/26-diagnostic-trees.js` (`const DIAGNOSTIC_TREES`): **13 trees covering all 11 Cardiology disease groups** that were in the old `CASE_GUIDES` (Acute Coronary Syndrome, Valvular Disease, Syncope, Resuscitation, Atrial Fibrillation, Arrhythmia ×2, Heart Failure ×2, Mitral Regurgitation, Myocarditis & Pericardial Disease, Cardiomyopathy, Pulmonary Embolism), 59 diagnosis/treatment outcomes total. Each tree is a recursive structure of decision nodes (`{q, clues?, options:[{label, next}]}`) terminating in leaf nodes (`{leaf:true, diagnosis, treatment:[...], notes?:[...]}`) — branching on the test/finding that actually separates a differential where one exists (e.g. ACS: ECG → troponin → symptom pattern separates STEMI/NSTEMI/UA/non-ACS), or on the situational factor that changes management where a group is really one disease staged by severity/risk (e.g. aortic stenosis: severity → symptoms → age/surgical risk).
  - Each answered step adds a card to the right, so the whole reasoning path stays visible; clicking any earlier step re-branches everything after it, with no separate "back" action needed. `treatment`/`notes` are short bullet fragments rendered as an icon checklist (check/warning icons) rather than prose, and a `clues` array renders as small recognition-tag chips under a question instead of a long parenthetical list — all to keep each card scannable at a glance.
  - **Navigation**: since dozens of trees are expected over time, the Cases tab now has a searchable nav sidebar organized **by chapter → disease group** (system → subject, e.g. Cardiology → Acute Coronary Syndrome), the same organizing principle used elsewhere in the app — not a flat dropdown. Selecting a tree starts it fresh in the detail pane; a "Restart this tree" control resets the current path without leaving the tree.
